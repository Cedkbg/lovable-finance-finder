1. Vue d'ensemble

RubriqueDétail
Nom du projetData Enricher Financier
Versionv4.2 (ENRICHER_v4.2)
TypeApplication web SaaS (Single Page Application)
Objectif principalTransformer un identifiant financier (ISIN, Ticker, CUSIP, SEDOL, FIGI) en un ensemble complet de données enrichies (RIC, MIC, secteur, devise, pays, etc.)
Public cibleAnalystes financiers, gestionnaires de portefeuilles, data managers, compliance officers
Stack techniqueReact 18 + TypeScript + Vite + Tailwind CSS + Lovable Cloud
API externeOpenFIGI v3 (Bloomberg Open Symbology)
AuthentificationEmail/mot de passe avec vérification email
Base de donnéesPostgreSQL (via Lovable Cloud)
Temps réelWebSockets (Supabase Realtime)
ExportXLSX (Excel) via bibliothèque xlsx

Ce logiciel permet d'enrichir automatiquement des données financières à partir d'identifiants standards de l'industrie. Il interroge l'API OpenFIGI de Bloomberg pour récupérer les métadonnées complètes d'un actif financier. Les données sont stockées en base, synchronisées en temps réel, et peuvent être exportées en Excel. Le système supporte l'import unitaire, par lot (bulk), et par pays (toutes les bourses d'un territoire).

6. Flux de données

Flux 1 : Recherche unitaire d'un actif

ÉtapeActionDétail
1Saisie utilisateurL'utilisateur entre un identifiant (ISIN, Ticker, etc.)
2Détection du typeLe système identifie automatiquement le format (ISIN=12 chars, SEDOL=7, etc.)
3Recherche DB localeVérifie si l'actif existe déjà en base de données
4Fallback mock dataSi pas en DB, cherche dans le jeu de données embarqué
5Appel OpenFIGISi toujours pas trouvé, appelle l'API via Edge Function
6EnrichissementL'Edge Function tente Mapping puis Search sur OpenFIGI v3
7Scoring & sélectionLe meilleur résultat est sélectionné (Common Stock > ETF > autres)
8PersistanceLe résultat est sauvegardé en DB (upsert sur ISIN)
9AffichageL'actif enrichi est affiché avec tous ses champs

Flux 2 : Import par pays

ÉtapeActionDétail
1Sélection paysL'utilisateur choisit un pays (ex: France)
2Résolution boursesIdentifie tous les codes boursiers du pays (France → FP, PA)
3Requêtes parallèlesL'Edge Function interroge chaque bourse avec pagination profonde
4DédoublonnageLes doublons sont éliminés par FIGI/compositeFIGI
5MIC fallbackSi OpenFIGI ne renvoie pas le MIC, il est déduit du code bourse
6Persistance bulkTous les actifs sont insérés en base (upsert)
7Notification liveLes autres utilisateurs connectés reçoivent une notification toast

Flux 3 : Synchronisation temps réel

ÉtapeActionDétail
1Connexion WebSocketAu chargement du Data Manager, un canal Realtime est ouvert
2Écoute INSERTLe canal écoute les événements INSERT sur financial_assets
3Écoute UPDATELe canal écoute les événements UPDATE sur financial_assets
4Écoute DELETELe canal écoute les événements DELETE sur financial_assets
5Mise à jour UILe tableau se met à jour automatiquement sans rechargement
6NotificationSi l'auteur est un autre utilisateur, un toast s'affiche

8. Glossaire — Termes clés

TermeAbrév.Définition
International Securities Identification NumberISINCode unique de 12 caractères identifiant un titre financier à l'échelle mondiale. Format: 2 lettres pays + 9 caractères alphanumériques + 1 chiffre de contrôle. Ex: US0378331005 (Apple Inc.)
Financial Instrument Global IdentifierFIGIIdentifiant global de 12 caractères créé par Bloomberg pour identifier de manière unique les instruments financiers. Format: BBG + 9 caractères. Ex: BBG000B9XRY4
Composite FIGIACFVersion agrégée du FIGI qui regroupe un même instrument coté sur plusieurs bourses sous un seul identifiant. Aussi appelé 'Aggregate Composite FIGI'.
Reuters Instrument CodeRICCode utilisé par Refinitiv (ex-Thomson Reuters) pour identifier un instrument sur une bourse spécifique. Format: TICKER.EXCHCODE. Ex: AAPL.UW
Market Identifier CodeMICCode ISO 10383 de 4 caractères identifiant une place boursière. Ex: XPAR (Euronext Paris), XNYS (NYSE), XNAS (NASDAQ)
Committee on Uniform Security Identification ProceduresCUSIPCode de 9 caractères utilisé principalement en Amérique du Nord pour identifier les titres financiers. Ex: 037833100 (Apple)
Stock Exchange Daily Official ListSEDOLCode de 7 caractères attribué par la London Stock Exchange pour identifier les titres au Royaume-Uni et en Irlande. Ex: 2046251
Ticker / SymbolTickerAbréviation alphabétique courte représentant un titre coté en bourse. Ex: AAPL (Apple), MSFT (Microsoft), BNP (BNP Paribas)
Exchange CodeexchCodeCode court Bloomberg identifiant une bourse. Ex: UN = NYSE, UW = NASDAQ, FP = Euronext Paris, LN = London SE
Row-Level SecurityRLSMécanisme de sécurité PostgreSQL qui restreint l'accès aux lignes d'une table selon des règles définies (ex: un utilisateur ne voit que SES données)
Edge FunctionEFFonction serverless exécutée côté serveur (Deno) servant de proxy sécurisé vers des APIs externes. Évite d'exposer les clés API côté client.
Realtime—Technologie WebSocket permettant la synchronisation instantanée des données entre le serveur et tous les clients connectés, sans rechargement.
Upsert—Opération combinant INSERT et UPDATE : si la ligne existe (conflit sur clé unique), elle est mise à jour ; sinon elle est créée.
SPASPASingle Page Application — Application web qui charge une seule page HTML et met à jour dynamiquement le contenu sans rechargement complet.
SDKSDKSoftware Development Kit — Ensemble d'outils et bibliothèques pour interagir avec un service.
Toast—Notification éphémère qui apparaît brièvement à l'écran pour informer l'utilisateur d'un événement (succès, erreur, info).
WebSocketWSProtocole de communication bidirectionnel persistant entre client et serveur, permettant les mises à jour en temps réel.
Common Stock—Action ordinaire — Titre de propriété dans une entreprise cotée donnant droit de vote et dividendes.
ETFETFExchange-Traded Fund — Fonds indiciel coté en bourse qui réplique la performance d'un indice (ex: S&P 500).
ETPETPExchange-Traded Product — Catégorie générale incluant les ETF, ETN et ETC cotés en bourse.
Market Sector—Classification du marché dans OpenFIGI: Equity (actions), Govt (obligations d'État), Corp (obligations d'entreprise), Mtge (hypothèques).
Bloomberg Open SymbologyBSYMInitiative de Bloomberg fournissant des identifiants financiers ouverts et gratuits via l'API OpenFIGI.
ISO 4217—Norme internationale définissant les codes alphabétiques de 3 lettres pour les devises. Ex: EUR, USD, GBP, JPY.
ISO 3166-1—Norme internationale définissant les codes pays à 2 lettres. Ex: FR (France), US (États-Unis), GB (Royaume-Uni).
ISO 10383—Norme internationale définissant les Market Identifier Codes (MIC) pour les places boursières.
